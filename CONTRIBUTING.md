# Contributing Guidelines

Thank you for your interest in contributing to this research platform.

## Branch Strategy

- `main` → stable
- `feature/*` → new features
- `docs/*` → documentation updates
- `chore/*` → infra/config updates

## Commit Convention

- `feat:` new functionality
- `fix:` bug fix
- `docs:` documentation
- `chore:` configuration/maintenance
- `refactor:` internal restructuring

Example:



feat: implement rule-based selector v1


## Pull Request Process

1. Create feature branch
2. Keep PR focused on single change
3. Provide description of:
   - Motivation
   - Design decision
   - Test steps

## Research Contribution

If contributing model improvements:
- Provide metric comparison (Rank Accuracy / NDCG@1)
- Specify dataset snapshot used
- Include reproducibility steps
